## What shipped

The 3.03 engine was running on `fixtures.ts`. This phase replaces them with the **real questions**, generated by code as typed, **language-neutral DATA** — no rendering (that is 3.05). Everything lives under **`src/content/tasks/`**.

**Seven procedural generators** (one per generated domain; attention is derived in scoring — no generator; no verbal generator — deferred):

| Domain | Paradigm (Прилог A) | What the generator emits |
|---|---|---|
| **Gf** | 3×3 matrices + number series (A.1) | a matrix (1–3 varying attributes, constant/progression rules, one hidden cell, one-attribute-mutation distractors) **or** a number series (rule order grows with level); select-one of 4 |
| **Gv** | mental rotation (A.2) | an asymmetric polyomino + 4 candidates (true rotation + mirror/other/wrong-angle distractors, all guaranteed distinct); select-one |
| **Gsm** | Corsi span (A.3) | a 6-tile board + a show-hide-repeat sequence; **forward/backward honored from `ItemFormat`**; `meta.spanLength` |
| **Gs** | timed symbol search (A.4) | an 18–28-cell field + target positions + a 20–25 s budget; `meta.cellCount`+`targetCount`; credit `(correct−0.5·false)/targets` net of penalty |
| **EF** | Tower of London (A.5) | start+goal states with a **BFS-exact** `minMoves`; `meta.minMoves`; credit = `minMoves/movesUsed` |
| **Glr** | paired-associate (A.6) | a learned pair set recalled over **several attempts** (a WeakMap learning block); `meta.attempt` feeds the learning slope |
| **CT** | sequencing/debug/loop/conditional/maze (A.8 + Дел 4) | 5 sub-types, each a typed spec + `judge`; a shared grid simulator validates sequence/maze solutions |

**The provider seam:** `createTaskItemProvider()` (`provider.ts`) implements the engine's `ItemProvider` + adds `getPracticeItem(domain)`. It is the production source 3.05 injects into `createDomainController` in place of `fixtures.ts` (**fixtures kept** for the engine's own unit tests). `getItem` dispatches to the domain generator; the engine's existing per-domain `deriveSeed` rng carries stream independence — the provider holds no state of its own.

**The content-spec contract (the 3.05 rendering seam, `types.ts`):** every `payload` is a `TaskSpec` discriminated on `taskType`, carrying `interaction` (the input model **as data**), a clearly separated `solution` (internal; `solution.answer` IS the canonical correct `Response.answer`), and the **exact `ItemScoringMeta`** v2 scoring reads. `shared.ts` provides the generic `correctAnswerFor`/`wrongAnswerFor` oracle (keyed off `interaction.mode`) that drives the tests + the end-to-end check. `glyphs.ts` holds the language-neutral token catalogs + polyomino geometry. `README.md` documents the seam, the determinism guarantee, and the PROVISIONAL parameter list.

**Determinism + fidelity, proven:** `prng.ts` only — **no `Math.random()`** anywhere (asserted by a source scan). The headline integration test drives a full session through the engine + the real provider + v2 scoring and gets a **byte-identical `SessionRun` + `CognitiveProfile`** across two runs; different seeds → different item sets (new sets on retest). Each `meta` was verified against what `scoring/v2/{raw,signals}` and `validity/flags` actually read. **No forbidden tokens** (score/IQ/%/rank/"level N") in any generated content.

## Surprises / off-spec

- **Spec-internal tension (not a plan mismatch):** Прилог A.3 fixes the Corsi board at **6 tiles**, but Прилог B.1 expects forward span up to **7** (age 13). Resolved by allowing **non-consecutive tile revisits** for spans > 6 (never two identical consecutive taps), flagged PROVISIONAL (#149) for the apparatus review. The PDF "wins" — we honored the 6-tile count and adapted the sequence rule.
- **135° rotation (A.2) is not representable** on the polyomino-on-grid model — deferred to 90° multiples (#148); would need vector-path shapes.
- **The Glr seam needed no change.** The brief pre-authorised a minimal additive seam extension "if genuinely needed." It was not: `ItemScoringMeta.attempt` already existed, and a stateful provider dispatch (the WeakMap block) is the smallest possible solution. Logged transparently as the one pure-function exception (#146).
- **One type-nesting bug** in the EF generator (a board `PegState[]` vs a list-of-boards `PegState[][]`) slipped past vitest — which strips types — and was caught by `tsc` in the gate, then fixed (runtime was already correct; tests were green throughout). The fresh-context review independently re-derived the BFS distances and confirmed EF correctness.
