The payoff: results, report & certificate

One family, three tones. Every surface is built on the locked v2 tokens, Montserrat, the five index hues and the puzzle-brain — the same spine the /test assessment already uses. The results screen is credible, the PDF is formal, the certificate is joyful. Each is the live mockup, embedded below.

The identity pentagon — not a gauge

The hero motif shared by the results screen and the PDF. The puzzle-brain the child fills during the test resolves into this: a whole, assembled, five-coloured shape. It is the deliberate replacement for the assessment kit's pentagon() radar chart, which encodes magnitude and is forbidden on these surfaces.

The hard rule. The pentagon is an identity graphic, not a measurement chart. Same shape, same size for every child. Only the five fixed index hues and the labels appear — it never grows, shrinks, fills part-way, or carries spokes/rings/axes. Two children with different results get the same shape; their words (on the cards) differ, never the geometry.
  • GeometryRegular pentagon, one vertex per index at its fixed axis angle; five kites, each centred on one vertex, filled with that index's --ix-* hue. White seams, ink outline.
  • Filter-freeFlat polygons + lines only — expresses 1:1 in @react-pdf/renderer (Svg/Polygon/Line). No gradients, no filters.
  • A11yGraphical object ≥3:1; carries an honest aria-label ("five areas, one whole shape"); meaning is never colour-only — every area is also a word on the cards.

Index card anatomy

  • NameFull index name in --ink-head bold.
  • BandThe level as a WORD in a hue-tinted pill ("Strongly developed") — never a number.
  • DescriptionOne plain-language line in --ink-muted.
  • ConfidenceClearly secondary: a neutral meta-label (high/medium/low) + a one-sentence note, below a divider. Reads as how sure the reading is — never a grade of the child.
  • ColourHue on the left rail + icon chip only. The word + label carry the meaning (colour-blind safe).

Surface B — the PDF report formal

Three A4 pages, the credibility document. No Bibi, no characters. Generous whitespace, strong Montserrat hierarchy, flat fills. Page 1 cover + pentagon + top strength; Page 2 the five indices in detail; Page 3 the narrative + next steps + the full disclaimer. Two optional photo slots that look intentional empty.

@react-pdf/renderer constraints honoured

  • LayoutFlexbox columns/rows only — no CSS grid, floats, or positioned overlays for content.
  • FillsSolid flat colour only. No gradients, no shadows, no filters.
  • PentagonThe filter-free vector (polygons + lines) — drops straight into react-pdf <Svg>.
  • FontsMontserrat must be embedded (Cyrillic + Latin) so the MK report renders; weights 400/600/700/800.

Page & type spec (print units)

  • PageA4 210×297mm, margins 15mm (10–12mm bottom).
  • Cover H130pt ExtraBold · section H2 15pt · block heads 11pt.
  • Body10.5pt / 1.6 · small & confidence note 8.5–9pt · footer 8pt.
  • Colour-codeEach index = 7mm hue rail + tinted icon + band-word pill in --ix-*-ink.

Guardrails — verified across all three

  • ✓No number anywhere — no IQ, %, score, percentile, rank, level, gauge, axis or bar. Bands are words; the pentagon is identity.
  • ✓Bibi only on the certificate — never on results or in the PDF. Placeholder only; never generated/redrawn.
  • ✓Bilingual MK + EN — Cyrillic default; every layout absorbs the text-length difference.
  • ✓Gender-neutral; the only child name is the optional browser-only one on the certificate.
  • ✓WCAG 2.2 AA — text ≥4.5:1, colour never the sole carrier, visible focus, ≥44px targets, reduced-motion, no anxious timers.
  • ✓PDF expressible in @react-pdf/renderer — flexbox, flat fills, filter-free vector, embedded Montserrat.
  • ✓Confidence is a neutral meta-label, always secondary to the band word.
  • ✗Never: a radar/spider pentagon, a progress bar, an emoji décor, a second typeface, a gradient.
